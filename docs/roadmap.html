우선 수정
1.1

에어마우스 (모션 센서) — 해결됨 ✓

완료 · 07-07
HTTPS
앱 :443 리스너가 LE 인증서(CN=pilot.cmdspace.work)로 서빙. DNS는 Tailscale IP(tailnet 전용=안전). https://pilot.cmdspace.work 무포트로 접속.
권한
iOS 17+는 설정 메뉴 없이 '팝업'만 → 요청을 pointerdown→click(iOS가 인정하는 제스처)으로 분리. 안내도 iOS 17+ 실제 모델로 정정.
방향
90° 어긋남 해소 — rotationRate.alpha(롤) 대신 gamma(yaw)를 수평에 사용 + 화면방향(세로/가로) 보정.
1.2

데스크탑(공간) 좌/우 전환 — macOS 26 차단

안 됨L
증상
4핑거 좌/우(데스크탑 전환) 기본 배정이 실제로 무동작.
근본 원인
macOS 26이 공간 전환을 합성 CGEvent(⌃←/→)·CGS/SkyLight 비공개 API 양쪽 다 차단 (EventInjector 주석에 확인됨). 프로그램적 전환 불가.
선택지
4핑거 좌/우를 실제 되는 액션(앱전환·뒤로/앞으로 등)으로 재배정, 또는 "미지원" 경고 노출. 미션컨트롤 ⌃↑·엑스포제 ⌃↓는 정상.
1.3

iPad 도킹 분할 버그

부분S–M
스플리터
드래그가 main padding + 14px 트랙을 빼지 않아 손가락과 어긋남. 그리드 내부 기준으로 재계산 필요.

터치타깃
스플리터 14px → 44px 가이드 미달, 잡기 어려움. hit 영역 20–24px로 확대.
코어
fr 계산·방향·클램프·회전·저장은 버그 없음으로 검증됨.
1.4

화면기록 권한 인앱 안내 없음

부분S
문제
미러/캡처는 손쉬운사용과 별개인 '화면 기록' 권한이 필요한데 메뉴바 UI는 손쉬운사용만 안내 → 권한 없으면 미러가 조용히 안 뜸.
필요
메뉴에 화면기록 권한 타일 + 설정 열기 링크, 최초 사용 전 프리플라이트 안내.

02

알려진 한계

설계상 / 현재 불가
트랙패드 연속 제스처 (손가락 추종·되감기)
시스템 트랙패드 드라이버가 멀티터치 위상(phase)을 넘겨야 함. CGEvent 키 주입으론 그 연속 이벤트 생성 불가(비공개 API) → 개별 단축키 발화가 최선.
불가
미러 오디오·비디오 코덱
현재 MJPEG intra-only(매 프레임 전체 JPEG). 오디오 없음, 동적 화면 대역폭↑. 고품질은 H.264 + WebCodecs/WebRTC 필요(대공사). 지금은 "보며 조작"용 경량 뷰어로 의도됨.
의도된 한계
공간 전환 프로그램 제어
위 1.2 — macOS 26 정책으로 합성·비공개 API 양쪽 차단.
OS 정책

개선하면 좋은 것

동작하지만
3·4핑거 제스처 사용성
인식은 되나 감(감도/오발화)이 완벽하진 않음. 재기준+정착창+우세축으로 개선했으나 실기기 튜닝 여지. "손가락 뗄 때 발화(중간 되돌리면 취소)" 옵션 검토.
부분S
cmux 브리지 결합도
소켓 인증 self-heal이 cmux 내부 JSON 스키마/에러 문자열에 강결합 → 업그레이드에 취약. 공식 automation-token API 나오면 이관.
동작M
Windows 포트 기능 매트릭스
Windows 서버엔 코어 입력만. mirror/capture/ocr/cmux/window 미구현이나 공유 클라가 탭을 띄움 → 미지원 탭 숨김/표기 + 매트릭스 문서 필요.
부분L
로그 로테이션 · 활성 포트 표기 · p12 패스프레이즈 외부화
/tmp 진단 로그 무제한 append · :80/:443 실패 시 조용히 스킵 · 인증서 패스('macpilot') 하드코딩.
동작S
카메라 OCR
실시간 카메라 아니라 파일 피커 1장. 빈 결과 무피드백 → 명시 메시지, (향후) 연속 스캔 모드.
