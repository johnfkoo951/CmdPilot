## 1. 지금 안 되는 것 (우선 수정)

### 1.1 🟢 에어마우스(모션 센서) — 해결됨 (2026-07-07)
> **완료**: `https://pilot.cmdspace.work`에서 동작 확인. 아래는 해결 경위(기록용).
> - HTTPS: 앱 `:443` 리스너가 LE 인증서(CN=pilot.cmdspace.work)로 서빙. DNS는 Tailscale IP(tailnet 전용=안전).
> - 권한: iOS 17+는 설정 메뉴 없이 '팝업'만 → 요청을 `pointerdown`→`click`(iOS가 인정하는 제스처)으로 분리.
> - 방향: `onAirMotion`이 `alpha`(롤) 대신 `gamma`(yaw)를 수평에 쓰도록 교정(90° 어긋남 해소) + 화면방향 보정.

<details><summary>원래 진단(참고)</summary>

#### (구) 에어마우스(모션 센서) — HTTPS 보안 컨텍스트 필요 · 난이도 M
- **증상**: 에어 버튼을 누르면 "http에선 모션 센서 차단" 안내가 뜨고 동작 안 함.
- **근본 원인**: iOS는 `DeviceMotion/DeviceOrientation`을 **secure context(HTTPS/localhost)에서만** 허용.
  현재 서버는 `http://<name>.local:8766` / `http://…ts.net:8766`(Tailscale)만 광고 → `window.isSecureContext=false` →
  `airStart()`가 즉시 차단(app.js). **모션→커서 코드(rotationRate + orientation 폴백 + 1€ 필터)와 권한 요청
  흐름(requestPermission)은 이미 올바르게 구현돼 있고, 오직 HTTP 오리진 때문에 도달을 못 할 뿐이다.**
- **이미 있는 것**: `HelperServer`에 조건부 `:443` TLS 리스너(`listener443`)가 존재
  (`App Support/MacPilot/tls/pilot.p12`, 패스프레이즈 `macpilot`, CN=pilot.cmdspace.work). 하지만 `updateURL()`이
  **무조건 http URL만** 만들어 메뉴/QR에 노출 → 인증서를 깔아도 http QR이 나와 에어마우스가 죽음.
- **해결안(택1)**:
  1. **(권장) `tailscale serve`** 로 로컬 서버 앞단에 Tailscale 발급 TLS를 붙여 `https://<mac>.<tailnet>.ts.net`
     제공 → 유효 secure context + wss 자동 승격(app.js는 https면 wss 선택). 그 https URL을 메뉴/QR에 광고.
     인증서·DNS 부담 없음.
  2. 기존 `:443` p12 경로 완성 — `listener443 != nil`이면 `httpURL`을 `https://pilot.cmdspace.work`로 바꿔 광고
     (`.local`은 LE 인증서 CN과 불일치하니 도메인 사용). DNS가 폰에서 맥으로 해석돼야 함.
  3. 스톱갭: `<name>.local`용 자체서명/mkcert 인증서 + 폰에 신뢰 설치 안내.
- **부수 작업**: 안내문을 "보안 컨텍스트(HTTPS)로 접속" 우선으로 재정렬(iOS 26/27은 사이트별 권한 거부가 아니라
  secure context 부재가 진짜 원인). 메뉴/QR에 "에어마우스용 https" 라벨.
- **파일**: `HelperServer.swift`(updateURL/loadTLSOptions), `MenuContentView.swift`, `app.js`(airStart 가드/안내).

</details>

### 1.2 🔴 데스크탑(공간) 좌/우 전환 — macOS 26에서 차단 · 난이도 L(우회 연구)
- **증상**: 4핑거 좌/우(데스크탑 전환) 기본 배정이 실제로는 무동작.
- **근본 원인**: `EventInjector.gesture()` 주석에 명시 — **macOS 26이 공간 전환을 합성 CGEvent(⌃←/→)와 CGS/SkyLight
  비공개 API 양쪽 모두 차단**. 즉 키 주입·비공개 API 어느 쪽으로도 프로그램적 공간 전환 불가.
- **영향**: 방금 넣은 `s4left/s4right = spaceLeft/spaceRight`가 이 맥에선 동작 안 할 수 있음.
- **선택지**:
  - 4핑거 좌/우를 macOS 26에서 실제 되는 액션으로 재배정(앱 전환/뒤로·앞으로/미션컨트롤 조합 등),
  - 또는 spaceLeft/spaceRight에 "이 macOS에서 미지원" 경고 노출,
  - 또는 미션컨트롤(⌃↑, 동작함)에서 공간을 눈으로 고르는 방식 안내.
- **참고**: 미션컨트롤 `⌃↑`·엑스포제 `⌃↓`는 정상 동작(네이티브 애니메이션 재생).

### 1.3 🟡 iPad 도킹 분할 버그 · 난이도 S~M

- **코어(fr 계산·방향·클램프 0.3~0.72·회전·localStorage)는 버그 없음**으로 검증됨.

### 1.4 🟡 화면기록 권한 인앱 안내 없음 · 난이도 S

  최초 사용 전 프리플라이트 안내.

## 2. 알려진 한계 (설계상/현재 불가)

- **트랙패드 연속 제스처(손가락 추종·되감기)** ⚫ — 시스템 트랙패드 드라이버가 멀티터치 제스처 위상(phase)을
  넘겨야 하는데, `CGEvent` 키 주입으론 그 연속 이벤트를 만들 수 없음(비공개 API 영역). → 개별 단축키 발화가 최선.
- **미러 오디오·비디오 코덱** 🟡 — 현재 MJPEG intra-only(매 프레임 전체 JPEG). 오디오 없음, 동적 화면에서 대역폭↑.
  고품질은 VideoToolbox H.264 + WebCodecs/WebRTC 필요(XL). 지금은 "보며 조작"용 경량 뷰어로 의도됨.
- **공간 전환 프로그램 제어** 🔴 — 위 1.2. macOS 26 정책.

## 4. 개선하면 좋은 것 (동작하지만)

- **3·4핑거 제스처 사용성** 🟡 — 인식은 되나 감(감도/오발화)이 완벽하진 않음. (재기준+정착창+우세축으로 개선했으나
  실기기 튜닝 여지). "손가락 뗄 때 발화(중간 되돌리면 취소)" 옵션으로 취소 가능성 부여 검토.
- **cmux 브리지 결합도** 🟢 — 소켓 인증 self-heal이 cmux 내부 JSON 스키마/에러 문자열에 강결합 → cmux 업그레이드에 취약.
  cmux 공식 automation-token API 나오면 이관. 1.8초 동기 sleep이 브리지 큐 점유.
- **Windows 포트 기능 매트릭스** 🟡 — Windows 서버엔 코어 입력만. mirror/capture/ocr/cmux/cterm/window/spotlight 미구현이나
  공유 웹 클라가 탭을 띄움 → 미지원 탭 숨김/표기 + 문서에 macOS/Windows 매트릭스 필요. 제스처 키코드 변환 검증.
- **로그 로테이션** 🟢 — `/tmp` 진단 로그가 무제한 append. 로테이션/비활성 옵션.
- **카메라 OCR** 🟢 — 실시간 카메라 아니라 파일 피커 1장. 빈 결과 무피드백 → 명시 메시지, (향후) 연속 스캔 모드.
- **활성 포트 표기** 🟢 — :80/:443이 실패해도 조용히 스킵, 메뉴엔 8766만. 실제 활성 포트 표기.
- **p12 패스프레이즈 하드코딩('macpilot')** — 외부화 검토.
